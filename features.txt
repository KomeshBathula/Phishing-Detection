PhishGuard AI - Implemented Features
======================================

1. DASHBOARD (Home Page)
   - Hero section with animated background highlights, scan-line animation, and gradient text
   - "Neural Engine Active" and "Zero-Trust Protocol" status badges
   - Two CTA buttons: "Initialize Scanner" (navigates to /analyzer) and "Enterprise Training" (navigates to /simulation)
   - Statistics cards displaying dummy data:
       * Total Scans: 1,247
       * Threats Detected: 389
       * Safe Content: 858
       * Detection Accuracy: 97.3%
   - Key Capabilities section listing: Multi-Format Analysis, Explainable AI, Security Training, Threat Intelligence
   - Recent Activity feed showing the last 5 scan history entries from mock data (7 items total in mock)
   - Each activity item shows: type icon (URL/Email/Text), input text (truncated), relative timestamp, and risk badge
   - Clickable activity items navigate to /history

2. THREAT ANALYZER PAGE (/analyzer)
   - Tabbed input for 3 content types: URL, Email, Text Message
   - Each tab has a custom placeholder and icon (Link2, Mail, MessageSquare)
   - Input validation:
       * URL tab: checks for valid URL pattern (https?://... or contains a dot)
       * Email tab: requires at least 20 characters
       * All tabs: checks for non-empty input
   - Pre-built sample inputs (clickable chips) for each tab:
       * URL: "Suspicious PayPal" (http://paypa1-secure.login-verify.com/account), "Fake Google Docs" (https://docs-google.share-document.net/view), "Legitimate Google" (https://www.google.com/search?q=weather)
       * Email: "Phishing Email" (Amazon spoofed email with urgency tactics), "Legitimate Email" (GitHub verification code)
       * Text: "Smishing Attack" (fake bank account locked SMS with .tk link), "Legitimate SMS" (UPS delivery notification)
   - Animated loading state during analysis with a rotating shield scanner and step-by-step scanning messages:
       * "Initializing AI engine..."
       * "Scanning for suspicious patterns..."
       * "Checking domain reputation..."
       * "Analyzing content semantics..."
       * "Cross-referencing threat database..."
       * "Generating risk assessment..."
   - Analysis uses a simulated 1.5-3 second delay to mimic server processing
   - Analysis engine (analyzer.js) uses keyword-matching and domain-checking logic:
       * Checks input against a list of 20 phishing keywords (urgent, suspended, verify, locked, etc.)
       * Checks URLs against 8 suspicious TLDs (.tk, .ml, .ga, .cf, .gq, .buzz, .top, .xyz)
       * Checks URLs against 14 known legitimate domains (google.com, github.com, paypal.com, etc.)
       * Returns mock analysis results based on matches (high/medium/low risk)
   - Results display section:
       * Animated circular progress ring showing risk score (0-100) with color coding (green/yellow/red)
       * Risk classification badge: "High Risk", "Medium Risk", or "Low Risk"
       * Dynamic headline: "Threat Signature Detected" / "Suspicious Activity Flagged" / "Security Clearance: Verified"
       * Quick stats: content type and number of findings
       * Score meter visual (Safe 0-30, Caution 31-70, Danger 71-100)
       * Algo version label: "ALGO_V5.2_MODEL_RF"
   - Explainability Panel (detailed analysis breakdown):
       * Content Analysis section with highlighted text (for emails/text): dangerous segments in red, warnings in yellow, safe in green, with hover tooltips explaining each flagged segment
       * Detection Findings section: list of reasons with severity icons (critical/high/medium/low/safe), keyword highlighting in each reason, severity badge per finding
       * URL Breakdown section (for URL analysis only): Protocol, Domain, Path, SSL Certificate status, Domain Age, Registrar, Suspicious Patterns list
       * "Why This Matters" advisory section with context-specific guidance and actionable steps based on risk level
   - Reset/Clear button to start a new analysis

3. PHISHING SIMULATION PAGE (/simulation)
   - Interactive quiz-style training with 6 pre-built scenarios from dummy data:
       * Scenario 1: Fake Microsoft sign-in alert email (phishing) - spoofed domain "micros0ft-security.com", 12-hour lockout threat
       * Scenario 2: Real Google security alert email (legitimate) - from accounts.google.com, personalized with user's name
       * Scenario 3: Fake Netflix billing SMS (phishing) - non-official domain "netflix-billing.support-update.com", 24-hour pressure
       * Scenario 4: Real LinkedIn search appearances email (legitimate) - from linkedin.com, personalized, links to official domain
       * Scenario 5: Real Apple iCloud storage SMS (legitimate) - links to official icloud.com
       * Scenario 6: Fake salary increase HR email (phishing) - generic domain, requests banking info, 48-hour deadline
   - Each scenario displays the full email/SMS content including From, Subject, and Body
   - Type badge indicator (Email or SMS) per scenario
   - Two answer buttons: "Safe" or "Phishing"
   - Progress bar showing completion (e.g., 3 of 6)
   - Live score tracker showing correct/total
   - After answering, reveals:
       * Correct/Incorrect banner with color coding
       * Whether it was phishing or legitimate
       * Detailed explanation of why
       * Red Flags section (for phishing scenarios) listing specific traps:
           - Spoofed Domain, Fear Tactic, Time Pressure, Generic Greeting, Fake Domain, Data Harvesting, Artificial Deadline, Fear of Loss
           - Each trap has a type label, the flagged text, and a description
   - Completion screen after all 6 scenarios:
       * Trophy icon with glow animation
       * Final stats: Intercepted (correct), Total Drills, Accuracy percentage
       * Performance message based on score:
           - 80%+: "Exceptional situational awareness..."
           - 60-79%: "Standard operational readiness achieved..."
           - Below 60%: "Security protocols breached. Remedial training recommended..."
       * "Re-initialize Drills" button to restart

4. SCAN HISTORY PAGE (/history)
   - Displays 7 mock scan history entries with: type icon, input text, type label, relative timestamp, risk score, risk badge
   - Search bar to filter history by input text (case-insensitive)
   - Type filter tabs: All, URL, Email, Text
   - Clickable entries open a detail modal
   - Detail modal contains:
       * Circular progress ring with the risk score
       * Risk classification badge
       * Input text and timestamp
       * Full ExplainabilityPanel (content analysis, detection findings, URL breakdown, advisory)
   - Empty state when no results match the filter/search
   - Mock history entries (dummy data):

       * Amazon spoofed email - Email - 91 - High Risk
       * google.com/search - URL - 12 - Low Risk
       * Bank locked SMS with .tk link - Text - 78 - High Risk
       * docs-google.share-document.net - URL - 52 - Medium Risk
       * GitHub verification email - Email - 8 - Low Risk
       * UPS delivery SMS - Text - 5 - Low Risk

5. SETTINGS PAGE (/settings)
   - "Data & Privacy Settings" header with gradient text
   - Local Storage Management card:
       * "Clear Scan History" button that clears chrome.storage phishing history
       * Loading spinner while clearing, success confirmation ("History Cleared")
   - Scheduled History Cleanup card:
       * Number input for auto-delete threshold (0-365 days)
       * 0 = disabled (history kept indefinitely)
       * Displays contextual message: disabled status or active purge interval
       * "Save Config" button with success feedback
   - Integrates with chrome.storage.local for persistence
   - Falls back gracefully when chrome.storage is unavailable (web context)

   - "System Active" status indicator with animated green pulse

   - Scroll-to-top on route change

   - Footer with: version label "SECURE_ENVIRONMENT_V1.4.2", copyright "2026 PhishGuard AI", Privacy Policy and Terms of Service links

7. BROWSER EXTENSION - POPUP (popup/Popup.jsx)
   - Extension header with PhishGuard AI branding
   - Global on/off toggle to enable/disable the extension
   - Theme cycling button: System, Dark, Light (with Sun/Moon/Monitor icons)
   - Active Tab Status section:
       * Displays the current tab's URL
       * Mock risk scoring: URLs containing "phishing" or "fake" get 80-99% risk, others get 0-19%
       * Shows status (Safe / High Risk / Unknown), risk percentage, and reason
       * Color-coded display (green for safe, red for danger)
   - Manual Analyzer section:
       * Textarea input for pasting URLs or text
       * "Scan Now" button with loading spinner animation
       * Displays scan result: status, score percentage, reason, and whether a URL was extracted
   - "Full Dashboard" button that opens the options/dashboard page
   - Disabled state shows "Protection is currently paused" with disabled UI

8. BROWSER EXTENSION - BACKGROUND SERVICE WORKER (background/background.js)
   - Initializes default settings on install (isExtensionEnabled: true, autoDeleteDays: 0)
   - Listens for tab URL changes (chrome.tabs.onUpdated)
   - Respects the global extension toggle (skips if disabled)
   - Auto-deletes history records older than the configured day threshold
   - Mock analysis: flags URLs containing "phishing" or "fake" as dangerous
   - Stores scan results in chrome.storage.local (capped at 100 entries)
   - Sends SCAN_COMPLETE message to popup
   - For suspicious URLs:
       * Sends SHOW_WARNING message to content script
       * Creates an OS-level notification with alert title and suspicious domain info

9. BROWSER EXTENSION - CONTENT SCRIPT (content/contentScript.js)
   - Listens for SHOW_WARNING messages from the background script
   - Injects a fixed red warning banner at the top of suspicious web pages
   - Banner displays: alert message, site type, risk score percentage
   - "Dismiss" button to close the banner
   - Uses inline styles (no external CSS dependency) with highest z-index
   - Prevents duplicate banners on the same page

10. REUSABLE UI COMPONENTS
    - Button: supports variants (primary, secondary, ghost, danger), sizes (sm, md, lg), loading state with spinner, icon support
    - Card: glass-morphism card with optional glow effect and hover state
    - RiskBadge: colored badge for High Risk (red), Medium Risk (yellow), Low Risk (green), supports sm and lg sizes
    - ProgressCircle: animated SVG circular progress indicator with color-coded risk score, configurable size and stroke width, eased animation using requestAnimationFrame
    - InputField: text input / textarea with icon, error message display, placeholder
    - Modal: overlay modal with backdrop blur, close button, scrollable content, sizes (sm/md/lg/xl), body scroll lock
    - LoadingScanner: animated scanner UI with spinning rings, pulsing shield icon, step-by-step scanning messages, shimmer progress bar
    - ExplainabilityPanel: composite component rendering highlighted content, detection findings, URL breakdown, and advisory actions
